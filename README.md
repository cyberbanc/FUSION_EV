# M9 FUSION EV — PancakeSwap Prediction paper bot

Новый общий бот объединяет пять групп сигналов и выбирает сторону по **ожидаемой прибыли (EV)**, а не только по вероятности победы.

## Что входит в решение

- **40% — Chainlink + движение текущего live-раунда PancakeSwap**: цена относительно `lockPrice`, импульс и ускорение по снимкам T−100…T−40.
- **15% — Binance microstructure**: движение 15/30/60 секунд, taker buy/sell, стакан, EMA 1m/5m. Если Binance недоступен, его вес автоматически становится нулевым.
- **20% — пулы PancakeSwap**: доля UP/DOWN, динамика доли и размер пула. Экстремальная толпа не копируется автоматически.
- **20% — Bayesian M9**: последние четыре результата + зона коэффициента. Используется сглаживание, поэтому 2 совпадения больше не дают ложные 100%.
- **5% — устойчивые паттерны длины 3–5**: минимум 20 совпадений и одинаковое направление в первой и второй половине истории.

Итоговая вероятность дополнительно сжимается к 50%, чтобы бот не создавал ложную уверенность.

## Выбор стороны

Для обеих сторон рассчитывается:

```text
EV = вероятность победы × ожидаемый чистый коэффициент − 1
```

Ожидаемый коэффициент берётся из текущих пулов, но уменьшается запасом `PAYOUT_HAIRCUT`, потому что распределение денег может измениться после T−40.

Бот всегда выбирает UP или DOWN. Если обе стороны имеют отрицательный EV, выбирается менее отрицательная сторона и минимальная ставка $5.

## Ставки

- первые `MIN_TRADES_FOR_VARIABLE_STAKE=100` закрытых сделок — только **$5**;
- EV ≥ 5% и согласие моделей ≥ 60% — **$10**;
- EV ≥ 12% и согласие моделей ≥ 70% — **$15**;
- банк ниже $400 — максимум $10;
- банк ниже $350 — только $5;

**Fibonacci и увеличение ставки после проигрыша отсутствуют. Автоматической остановки по размеру paper-банка нет.**

## Режим

Эта версия — **только paper trading**. Она не содержит приватного ключа и не отправляет реальные транзакции. Сначала необходимо собрать минимум 500–1000 новых раундов и проверить EV, profit factor и просадку.

## Установка GitHub → Railway

1. Создайте новый пустой GitHub repository.
2. Загрузите **содержимое этой папки** в корень репозитория.
3. В Railway создайте новый Project → **Deploy from GitHub repo**.
4. В этом же проекте добавьте **PostgreSQL**.
5. Откройте сервис бота → Variables → Raw Editor.
6. Откройте отдельный пакет **M9_FUSION_EV_RAILWAY** и скопируйте содержимое `RAILWAY_VARIABLES.txt`.
7. Замените `https://YOUR-BSC-RPC-ENDPOINT` на рабочий HTTPS RPC BNB Smart Chain.
8. Убедитесь, что `DATABASE_URL=${{Postgres.DATABASE_URL}}` ссылается на правильное имя PostgreSQL-сервиса.
9. Используйте **одну Railway replica**.

`PORT` Railway добавляет автоматически.

### Важные статусы `/signal`

- `WAIT` — ещё не наступило T−40;
- `LOCKED` — решение записано в PostgreSQL и больше не изменится;
- `decision_quality=WARMUP_*` — период первых 100 сделок с фиксированной ставкой $5;
- `FORCED_MINIMUM` — обе стороны имеют отрицательный EV, выбрана менее плохая.

## История

PostgreSQL хранит все решения без автоматического удаления. Ограничение применяется только к конкретному API-запросу через `limit`.

Полная история через `/status` и скачивание JSON:

```text
/status?history=all&limit=100000
```

При прямом открытии этой ссылки браузер получает файл `m9_fusion_ev_history.json`. Внутри ответа также есть `history_count`, `history_returned`, `history_has_more` и `history_next_offset`. Если записей станет больше 100 000, следующую часть можно получить через `offset`, например:

```text
/status?history=all&limit=100000&offset=100000
```

Обычный компактный статус по-прежнему доступен так:

```text
/status?limit=30
```

CSV всей доступной истории:

```text
/history/export.csv
```

## Почему `BOOTSTRAP_LOOKBACK=160`

При первом запуске бот загружает последние 160 закрытых раундов для первоначальной M9/паттерн-статистики. После запуска база непрерывно дополняется и не удаляет старые раунды. Слишком большой bootstrap создаёт сотни RPC-запросов при старте и может вызвать timeout/429.

## Безопасность и ограничения

- нет гарантий прибыли;
- Binance используется только как дополнительный источник, так как итог PancakeSwap определяется Chainlink;
- при недоступности Binance бот продолжает работу;
- при недоступности всех BSC RPC решение не создаётся, чтобы не записать данные с выдуманной ценой;
- реальные транзакции намеренно не включены в первую тестовую версию.
