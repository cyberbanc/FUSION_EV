M9 FUSION EV 1.3.6 — ОСНОВА НАСТРОЕК
====================================

Пакет реализует утверждённую схему, выбранную после анализа полной выгрузки ev_history.csv:

MIN_TRADE_EV=-0.06
EV < 0      -> $5
0 <= EV < .05 -> $10
EV >= .05   -> $15
SHADOW_RECENT_WINDOW=8
SHADOW_RECENT_MIN_PNL=-30
QUALITY_WINDOW=30
QUALITY_MIN_SAMPLES=10
QUALITY_MIN_WIN_RATE=0.45
QUALITY_MIN_PROFIT_FACTOR=0.85
3 реальных LOSS подряд -> пропуск 1 следующего решения

Период исходной аналитической выборки:
14 июля 2026, 17:52:46 (ОАЭ)
по 20 июля 2026, 11:25:53 (ОАЭ)
1614 закрытых раундов, epoch 498252–499865.

Исторические результаты являются подбором параметров на прошлой выборке и не гарантируют будущий PnL.
Реальный worker использует только уже подтверждённые результаты; из-за задержки settlement точный live-порядок включения фильтров может отличаться от упрощённого табличного расчёта.

Торговая логика версии 1.3.6.3 не изменена. Исправление касается только совместимости Web3.py с BNB Smart Chain PoA/PoSA extraData.
