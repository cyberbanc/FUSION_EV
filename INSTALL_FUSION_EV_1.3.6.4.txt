M9 FUSION EV 1.3.6.4 — DB GET_DECISION HOTFIX

УСТАНОВКА
1. Распакуйте ZIP.
2. Полностью замените содержимое текущего GitHub-репозитория файлами из пакета.
3. В Railway откройте Variables и примените значения из RAILWAY_VARIABLES_FUSION_EV_1.3.6.4.txt.
4. Обязательно установите MODEL_VERSION=1.3.6.4.
5. Не удаляйте PostgreSQL и не меняйте DATABASE_URL.
6. Оставьте одну Railway replica и выполните Redeploy.

ПРОВЕРКА ПОСЛЕ DEPLOY
1. /healthz должен вернуть HTTP 200 и version=1.3.6.4.
2. /health должен показать connected=true и database_connected=true.
3. worker.last_tick.ok должен стать true.
4. В worker.last_tick не должно быть AttributeError: module 'app.db' has no attribute 'get_decision'.
5. /signal должен быстро возвращать betting_epoch, seconds_to_lock и snapshot.
6. Tilda должна показывать текущий раунд без изменения своего кода.

ВАЖНО
- Торговая логика не изменена.
- Ставки остаются $5 / $10 / $15 по selected EV.
- Shadow-фильтр, quality-фильтр и cooldown не изменены.
- Банк, PnL и вся PostgreSQL-история сохраняются.
